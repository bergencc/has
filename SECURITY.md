# Security Policy

## Supported versions
Security fixes are applied to the latest main branch. Older versions are not maintained unless explicitly stated.

## Reporting a vulnerability
If you believe you have found a security issue, please report it privately.

Email: `ghost@snabxo.com`

Please include:
- A clear description of the issue
- Steps to reproduce
- Impact assessment (what data or systems are affected)
- Any proof-of-concept (if safe to share)

## Coordinated disclosure
We will acknowledge reports within 72 hours and provide a status update within 7 days. We may request additional details or a safe reproduction environment.

Please do **not** open a public issue or discussion for security concerns.

## Safe harbor
We will not pursue legal action against researchers who:
- Act in good faith
- Avoid privacy violations, data destruction, or service disruption
- Provide us reasonable time to remediate before public disclosure

## Out of scope
The following are generally out of scope:
- Social engineering or phishing of project members
- Physical attacks or threats
- Denial-of-service attacks
- Vulnerabilities in dependencies without a demonstrated impact on this project

## Data sensitivity
This is a student-centered project. Avoid using real student data in tests or reports. If sensitive data is discovered, notify maintainers immediately and do not retain copies.
